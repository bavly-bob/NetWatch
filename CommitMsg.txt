feat(networking): implement async TCP client and improve Windows compatibility

Introduce a fully functional asynchronous TCP client built on Boost.Asio, completing the core client-side networking layer.

Key changes:
- Client implementation:
  - Implemented async connection flow using resolver + async_connect
  - Added connection lifecycle management with disconnect handler
  - Enabled safe message sending with connection state validation
  - Exposed access to active connection object
- Refactor:
  - Renamed internal client members (io_context, resolver, connection) for improved clarity and scalability
- Build system:
  - Defined _WIN32_WINNT=0x0601 for Windows builds to ensure compatibility with Boost.Asio networking features
- Testing:
  - Added basic integration test for client connection using Boost.Asio
  - Introduced minimal HTTP test server (Winsock-based) for manual testing
- Cleanup:
  - Removed obsolete CommitMsg.txt file

Notes:
- Current DNS resolution is synchronous and may be migrated to async_resolve in future iterations
- Test server is Windows-specific and intended for temporary validation only
